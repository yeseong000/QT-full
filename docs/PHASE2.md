# PHASE — UI/UX 개편 (인덱스 + 캘린더 통합)

> 주만나 AI 큐티 — 2026-05-08 시작 · v0.1
> 이 문서는 대화 간 컨텍스트 연속성을 위한 핸드오프 문서.

---

## 1. 개편 목적

**문제 인식**
- 묵상 달력 / 묵상 기록(archive) / 날짜 / 스트릭이 화면 곳곳에 흩어져 있어 인지 부담이 크다.
- 책 아이콘이 OS 기본 톤이라 디자인 일관성이 깨지고, 발견성도 낮다.
- "N일 연속"이라는 표현이 큐티 본질(하나님과의 지속적 관계)과 어울리지 않는다.
- 초신자에게 앱을 권유할 때 URL을 복붙하는 마찰이 크다.

**개선 방향 (한 줄 요약)**
- 인덱스는 더 가볍게, 캘린더는 더 깊게. QR로 전파 마찰 0.

## 2. 결정 사항 (확정)

### 2-1. 인덱스 헤더

| 영역 | 변경 전 | 변경 후 |
|---|---|---|
| 좌측 | `🌱 N일 연속` 알약 (0이면 숨김) | `26년 5월 8일 (금) · N회 완료` 알약 (0회면 `26년 5월 8일 (금)`만) |
| 우측 | 책 아이콘 + 설정 | **QR 아이콘 + 설정** (책 아이콘 제거) |
| 인사말 아래 날짜 | `2026년 5월 8일 금요일` | **제거** (날짜는 좌측 알약으로 이동) |
| 알약 클릭 시 | `archive.html` | **`calendar.html`** (단일 진입점) |

### 2-2. 시간대 매핑 (변경)

```js
function getTimeSlot(hour) {
  if (hour >= 4  && hour < 7)  return 'dawn';      // 04~07
  if (hour >= 7  && hour < 12) return 'morning';   // 07~12
  if (hour >= 12 && hour < 17) return 'afternoon'; // 12~17 (변경: -1h)
  if (hour >= 17 && hour < 22) return 'evening';   // 17~22 (변경: -1h, -1h)
  return 'night';                                  // 22~04 (변경: -1h)
}
```

| 시간대 | 인사말 (그대로) |
|---|---|
| 새벽 04~07 | 거룩한 새벽이네요 |
| 아침 07~12 | 은혜로운 아침이에요 |
| 오후 12~17 | 평안한 오후예요 |
| 저녁 17~22 | 온유한 저녁이네요 |
| 밤 22~04 | 고요한 밤입니다 |

### 2-3. QR 공유 모달

| 요소 | 결정 |
|---|---|
| 제목 | `큐티 묵상` |
| 부제 | `QR을 스캔하면 사이트로 이동해요.` |
| QR 크기 | 240×240 (16px 흰 여백 포함) |
| QR 색상 | 순수 검정/흰색 (인식률 우선, 가운데 로고 없음) |
| URL 영역 | 베이지 배경 + "복사" 텍스트 버튼 |
| 우상단 X 버튼 | 추가 (다른 사람에게 보여주는 상황 고려) |
| 시트 형태 | 기존 `modal-sheet` 재사용 + swipe-down |

### 2-4. 캘린더 화면 (archive 흡수)

| 항목 | 변경 |
|---|---|
| 배경 | 시간대별 PNG + `filter: blur(50px) saturate(0.85)` + `transform: scale(1.1)` |
| 오버레이 | 낮시간대 `rgba(255,255,255,0.45)` / 밤 `rgba(0,0,0,0.4)` |
| 카드 스타일 | 반투명 흰색 `rgba(255,255,255,0.85)` + `backdrop-filter: blur(8px)` (글래스모피즘) |
| 다크 모드 (밤) | 반투명 흰색 `rgba(255,255,255,0.12)` + 흰 보더 + 흰 텍스트 |
| 인라인 미리보기 | archive 디테일 전체 흡수 (핵심 요약 / 마음 / 밑줄 / 느낀 점 / Q&A) |
| "전체 기록 보기" 링크 | **삭제** |
| 미선택 안내 | `🌿 점이 있는 날을 누르면 그날 묵상을 다시 볼 수 있어요` |
| `archive.html` | **삭제** (calendar.html이 단일 진입점) |

### 2-5. 통계 아이콘 (이모지 → SVG)

**총 묵상 횟수** — 4단계 성장 시스템

| 단계 | 회수 | 명칭 |
|---|---|---|
| 1 | 0~6회 | 씨앗 (떡잎 2장) |
| 2 | 7~13회 | 새싹 (잎 1쌍) |
| 3 | 14~20회 | 성장기 (잎 2쌍) |
| 4 | 21~27회 | 성장기 + 꽃봉오리 |
| 5 | 28회+ | **개화** (잎 3쌍 + 꽃) — 이후 그대로 유지 |

**연속일** — 잎 한 장 SVG (B-1)

### 2-6. 캘린더 셀 상태

| 상태 | 표시 |
|---|---|

| 완료한 날 | 숫자 + 세이지 점 (`#5BA892`) |

## 5. 핵심 학습 (이번 라운드)

- **연속성보다 누적** — 큐티는 시험 공부가 아니라 동행이므로 "N일 연속"보다 "N회 완료"가 부담이 적다.
- **글래스모피즘** — iOS 잠금화면 위젯 톤. PWA 타겟에서 유효함.
- **archive 분리는 인지 부담** — 같은 목적(되새김질)을 두 화면이 나눠 가지면 사용자 동선만 늘어남.
- **QR 가운데 로고 X** — 인식률 우선. 토스/카카오페이/인스타 모두 동일 원칙.
- **이미지 DPR 3 대응** — iPhone 14 Pro의 393px CSS 픽셀 = 1179px 물리 픽셀.
