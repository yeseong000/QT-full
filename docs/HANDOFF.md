## 🔥 자주 묻는 질문

### Q. 오륜교회 사이트에서 데이터 못 가져올 때?
→ 403 에러는 User-Agent 차단. `scripts/fetch_qt.py` 헤더 조정.
→ GitHub Actions 환경에서는 대부분 정상 동작.

### Q. AI가 이상한 답변 주면?
→ `docs/AI_PROMPT.md`의 프롬프트 조정 → `scripts/generate_ai.py` 반영

### Q. iPhone에서 다이나믹 아일랜드에 UI가 가려요
→ Safari 일반 모드에서는 가릴 수 있음
→ "홈 화면에 추가" 후 **PWA 모드로 실행**하면 정상

### Q. 새 기능 추가하고 싶어요
→ 먼저 `docs/history/NEXT_STEPS.md` Phase 6 확장 기능 섹션 확인
→ 없으면 `docs/ROADMAP.md`에 추가 후 구현

### Q. 디자인 바꾸고 싶어요
→ `docs/DESIGN_TOKENS.md`와 `src/preview.html`로 먼저 시각화
→ 변경 이력 남기기

## 💬 Claude 웹과의 대화 팁 (중요)

사용자가 Claude Code(로컬 CLI)가 아닌 **Claude 웹**으로 작업할 때 효율을 높이는 방법입니다.

### ✅ 첫 메시지 템플릿

```
안녕, 이 프로젝트를 이어서 작업하려고 해.
첨부한 BRIEFING.md를 먼저 읽어줘.
이후에 내 구체적인 질문을 기다려줘.
이해했으면 "이해 완료"라고만 답해.
```

→ 이렇게 하면 AI가 컨텍스트를 먼저 소화한 뒤 작업을 시작합니다.

### ✅ 파일 첨부 전략

Claude 웹은 파일 업로드가 가능합니다. 작업 성격별로:

| 작업 | 첨부할 파일 |
|---|---|
| 전체 인수인계 | `BRIEFING.md` |
| 새 화면 제작 | `BRIEFING.md` + `docs/USER_FLOW.md` + `src/step-1-scripture.html` (참고용) |
| 디자인 수정 | `BRIEFING.md` + `docs/DESIGN_TOKENS.md` + `src/styles/common.css` |
| 크롤러/AI 수정 | `BRIEFING.md` + `docs/SCRAPER.md` + 해당 Python 파일 |
| 일본어 현지화 | `BRIEFING.md` + `JAPANESE_LOCALIZATION_PLAN.md` + 해당 파일 |

### ✅ 작업 요청 공통 규칙 (사용자 선호)

매 대화에서 AI에게 상기시킬 원칙:

1. **수정 전에 원인/방향을 표(table)로 먼저 설명**
2. **수정안은 파일 단위로 구분**해서 제시 (한 번에 여러 파일 섞지 않기)
3. **디자인 토큰과 금지사항을 반드시 지킬 것**
4. **한국어로 답변**, 정중한 말투
5. **불필요한 설명 반복 금지** — 핵심만 간결히

### ⚠️ Claude 웹의 한계

- **컨텍스트가 유한함** → 긴 대화는 핵심만 요약해서 새 대화로 이어가기
- **로컬 파일 직접 수정 불가** → AI가 수정안을 주면 사용자가 직접 복붙/저장
- **코드 실행 불가** → 실행/테스트는 사용자 로컬에서
